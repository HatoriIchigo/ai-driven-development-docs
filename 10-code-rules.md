# コーディングルール

## アーキテクチャ

より多層にしたLayered Architectureを使用する。

**Top層**、**Service層**、**Repository層**、**DTO層**に分かれる

## Top層

- ルーター、トップレベル。
- ここで条件分岐は行わない

## Service層(LayerN層)

- ここで処理を記載する
- top層の解釈及びRepositoryの合成
- 1ファイル=1クラス
- 1ファイルは500行以内、1メソッド100行以内
- 同種レイヤーをより下層から呼び出すことは不可
    > **クラス**: Aクラス(Layer 3)、Bクラス(Layer 2)、Cクラス(Layer 4)
    > **関係**: Aクラス --> Bクラス
    > この際、CクラスはAクラスは呼び出せるが、Bクラスは呼び出し不可
- 同じ構成のレイヤーを呼び出す際にはレイヤーを分割してLayerを昇格する
    > **クラス**: Aクラス(Layer 2)、Bクラス(Layer 2)、Cクラス(Layer 2)、Dクラス(Layer 3)、Eクラス(Layer 3)
    > **関係**:
    > - Dクラス --> Aクラス、Bクラス、Cクラス
    > - Eクラス --> Aクラス、Bクラス
    > 
    > この場合、DクラスはLayer 4へと昇格し、Aクラス、Bクラスへのアクセスは禁止。
- かぶる構成のレイヤーがある場合は分割してLayerを昇格する
    > **クラス**: Aクラス(Layer 2)、Bクラス(Layer 2)、Cクラス(Layer 2)、Dクラス(Layer 2)、Eクラス(Layer 3)、Fクラス（Layer 3）
    > **関係**:
    > - Eクラス --> Aクラス、Bクラス、Cクラス
    > - Fクラス --> Bクラス、Cクラス、Dクラス
    > 
    > この場合、Bクラス、Cクラスがかぶるため、BクラスCクラス制御用にGクラス(Layer 3)を作成。Eクラス、Fクラスは昇格して(Layer 4)になり、Gクラスを使用するようにする
- 親子関係のレイヤーで差が2以上の場合はLayerを降格する
    > **クラス**: Aクラス(Layer 1)、Bクラス(Layer 3)
    > **関係**: Bクラス --> Aクラス
    > 
    > この場合、BクラスとAクラスに2以上の差があるため、BクラスをLayer 2に降格
    > なお、基準は下のレイヤーが持つ

## Repository層

- DTOを利用した入出力を定義。（外部接続、DB操作など）
- 条件分岐や繰り返しは行わない
- 合成も行わない
- 基本はDTOのin/outと1対1対応になる
- 検索条件の組み立てはService層で行う

## DTO層

- Lombokを利用したDTO定義
- DTOを定義する。
- ハードコーディング不可、ifによる条件分岐不可
- ディレクトリでin(リクエスト)/out(レスポンス)を分ける

## その他

- Constantとしてハードコード置き場を作成。ここ以外にハードコードは不可
- logにはログ用のハードコードを作成
- 戻り値をconstant化の禁止

## Layer違反の自動検出

Claude Code の hooks（Write/Edit時）で違反を機械的に検出・Rejectする。

- **解析方法**：LSP / AST によるimport・依存関係の静的解析
- **Layer番号の判別**：ディレクトリパス（`service/layerN/`）から自動取得
- **検出フロー**：
    1. 対象ファイルのパスからLayer番号を取得
    2. AST解析でimport/依存クラスを抽出
    3. 依存先のLayer番号と比較
    4. 違反（同レイヤー横断・下層から上層への参照）→ Reject
- **かぶり検出**：LSPの`findReferences`でプロジェクト全体の参照グラフを構築して判断
